# Guardrails

## Product
- Keep N5 as the main product experience until it feels polished
- Do not add random features that dilute beginner learning
- Prefer incremental upgrades over broad rewrites

## UX
- Every feature must work well on mobile
- Keep the path to first study action short
- Celebrate progress often, but do not interrupt core learning too much
- Prioritize clarity over visual cleverness

## Content
- Keep beginner flows approachable
- Do not mix advanced content into core N5 journeys
- Make feedback immediate and encouraging

## Engineering
- Preserve existing app/router/store patterns unless there is clear product benefit
- Reuse current learning, progress, SRS, and onboarding systems where possible
- Prefer additions that fit existing structure in `src/app`, `src/components`, `src/store`, and `src/data`

## Decision Filter
- When choosing between options, prefer the one that makes study sessions more motivating, more understandable, and easier to repeat daily
